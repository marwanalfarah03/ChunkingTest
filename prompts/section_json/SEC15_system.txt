Section: SEC15 — الملفات ومدد الحفظ (Files & Retention Periods)

This section specifies document archival requirements, retention durations, and storage obligations. It appears in two distinct formats:

Format A — Simple (single CL reference): A minimal two-row table with a full-width heading row "الملفات ومدد الحفظ" followed by a single content cell (CL reference). This is the most common format.

Format B — Retention table (2-column): A table with the heading row, then a column-header row, then multiple data rows listing file categories and their retention periods. The two columns are "الفئة" (Category) and "اسم الملف" (File Name). Column headers may be colored and appear as CL IDs in the rendered output; use the column header inspection result to get the resolved column labels.

Canonical column order (from column header inspection, when applicable): ["الفئة", "اسم الملف"]

Note: In some documents, SEC14 and SEC15 share the same chunk. Focus only on the SEC15 block (starting from the heading for this section).

---
INPUT FORMAT

You receive:
1. Extracted section content — rendered table text, possibly covering multiple sections.
2. The target section ID: SEC15.
3. Column header inspection result for SEC15 (passed as context, if available): the resolved valid_column_order and actual_header_cell_ids.

The content may contain:
- Multiple sections merged together. Focus only on SEC15.
- The section heading follows the pattern "N. <title> [#xxxxxx]". The title may be the full name "الملفات ومدد الحفظ" or an abbreviated/variant form (e.g., "مدد الحفظ", "الملفات", "Files & Retention", etc.). Identify the SEC15 block by its position in the document and its structural match to one of the two formats below — do NOT require an exact match to the full section name.
- For Format A: one content cell (CL reference).
- For Format B: a column-header row followed by multiple data rows.
- Light gray or white column headers (color codes like #D9D9D9, #E7E6E6) indicating the retention table.
- Color codes, HTML tags, ASCII borders.
