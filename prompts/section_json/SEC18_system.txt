Section: SEC18 — التعريفات والمختصرات والمصطلحات (Definitions / Terms / Abbreviations)

This section provides a glossary of domain-specific terms, banking acronyms, and abbreviations. It appears in two formats:

Format A — Simple (single CL reference): A minimal two-row table with a full-width heading row (various label forms, see below) followed by a single content cell (CL reference). This is the most common format.

Format B — Glossary table (multi-column): A table with the heading row, then a column-header row, then multiple definition rows. Each row has a term and its definition.

The heading label varies across documents:
- "التعريفات والمختصرات والمصطلحات المستخدمة"
- "التعريفات والمختصرات والمصطلحات"
- "التعريفات"
- "التعريفات والمختصرات"
All are SEC18.

Note: In some documents, SEC18 and SEC10 (المراجع الرئيسية) share the same cell. In that case, extract the combined content as SEC18. Additionally, SEC18 occasionally appears before SEC09 or after SEC10 depending on the document's structure.

---
INPUT FORMAT

You receive:
1. Extracted section content — rendered table text, possibly covering multiple sections.
2. The target section ID: SEC18.

The content may contain:
- Multiple sections merged together. Focus only on SEC18.
- The section heading in one of the label forms listed above: "N. التعريفات والمختصرات والمصطلحات المستخدمة [#xxxxxx]"
- For Format A: one content cell (CL reference).
- For Format B: a column-header row followed by multiple definition rows.
- Color codes, HTML tags, ASCII borders.

---
OUTPUT FORMAT

Format A — Simple:
{
  "section_id": "SEC18",
  "section_heading": "<heading text as it appears>",
  "format": "simple",
  "content": "<CL reference or visible text; multiple values joined with \\n if more than one>"
}

Format B — Glossary table:
{
  "section_id": "SEC18",
  "section_heading": "<heading text as it appears>",
  "format": "table",
  "columns": ["<col1>", "<col2>", ...],
  "header_cell_ids": ["CL000...", ...],
  "rows": [
    {"<col1>": "<value>", "<col2>": "<value>"}
  ]
}

Rules:
- Use Format A when there is a single content cell after the heading.
- Use Format B when the section has a column-header row and multiple definition rows.
- For Format A: if additional text appears after the main content cell that belongs to this section (not from another section), append it to `content` separated by `\n`.
- For Format B: "columns" lists column labels (visible text or CL IDs if hidden); "header_cell_ids" lists the actual header cell IDs if present, otherwise [].
- For Format B: if additional text appears that doesn't fit into a table row, include it in an optional top-level `"additional_text"` field (multiple pieces joined with `\n`).
- If a column header is a CL ID, use that CL ID as the key in each row — do NOT repeat it as a value.
- Preserve all cell IDs (CL000XXX) exactly.
- Preserve text exactly including color codes and HTML tags.
- Do NOT include content from SEC10 (المراجع الرئيسية) or SEC11 (التعليمات العامة).
- If SEC18 content is not found in the input, return {"section_id": "SEC18", "status": "not_found"}.
- Return JSON only — no prose, no explanation, no markdown wrapper.

Example output (Format A):
{
  "section_id": "SEC18",
  "section_heading": "4. التعريفات والمختصرات والمصطلحات المستخدمة [#253744]",
  "format": "simple",
  "content": "CL000104"
}
