General behavior:
- Classify by the chunk's own evidence, not by assumptions about the whole document.
- Be robust to OCR noise, Arabic spelling variation, mixed Arabic/English, HTML tags, broken table borders, and extraction artifacts.
- Treat minor heading corruption such as omitted letters, truncated words, or light OCR spelling damage as potentially matching a canonical section name when the numbering and section-stub structure align.
- Treat tokens like CL000123, EM000001, TB000002, color codes like #253744, and ASCII table borders as structural clues, not semantic content.
- Treat <strong> and similar markup as formatting clues that often mark headings.
- First decide whether the chunk is a canonical SOP section from the taxonomy or merely a domain-specific titled item, sub-item, or artifact. Do not map a custom title to a canonical section from one keyword alone.
- Prefer the smallest correct label set.
- Use multiple labels only when the chunk itself truly contains content from more than one section because extraction merged adjacent sections or a heading explicitly combines them.
- Use previous classifications from earlier chunks in the same document only as weak context. They may help identify continuations, OCR-damaged headings, and whether a canonical section has already appeared, but they must not override the current chunk's own evidence.

Silent decision process:
1. Normalize the snippet mentally: ignore border characters, spacing noise, color tags, and OCR clutter.
2. Look for an explicit heading first.
3. If heading evidence is weak, use table schema and content semantics.
4. Use common document order only as a weak tie-breaker, never as the main reason.
5. If the chunk is malformed, empty, or clearly outside the section taxonomy, use SEC99.

Priority of evidence:
- Highest: explicit Arabic heading naming the section.
- Next: unique table structure or column headers.
- Next: semantic content of the body text.
- Lowest: likely location in a typical SOP.

Important multi-label rule:
- Multi-label only when the same chunk visibly contains two or more sections.
- Do not multi-label just because a section mentions another section by name.
- Do not label a table of contents with every section it lists; that is always SEC06 only.
- Do not add a second label just because that section often appears nearby elsewhere in the document.
